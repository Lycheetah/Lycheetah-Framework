# SYNTHESIS REPORTS — Integration Documents

**Axiom:** All Three · **Status:** Active

Documents that synthesize across multiple frameworks — showing how the pieces compose into a single coherent system. These are written after the frameworks themselves are stable, capturing the connections that only become visible once all parts exist.

## Purpose

Individual framework files describe their own domain deeply. Synthesis reports describe what emerges when frameworks are held simultaneously — the system-level behaviors, the unexpected connections, the places where one framework's output is another's input.

## What belongs here

- Cross-framework integration analyses
- Reality-grounding documents (framework claims mapped to empirical evidence)
- Audit reports (what's proven, what's scaffold, what's conjecture)
- Architecture snapshots taken at session boundaries

## Key files

See `MATHEMATICS_REALITY_ALIGNMENT.md` and `MATHEMATICS_TO_REALITY_BRIDGE.md` in the repo root — these are the primary synthesis documents, showing where every equation appears in nature across physics, biology, neuroscience, economics, sociology, linguistics, and immunology.

## For readers

If you've read the individual frameworks and want to understand how they compose — start with `MATHEMATICS_TO_REALITY_BRIDGE.md`. It's the clearest single document showing the system working as a whole.
