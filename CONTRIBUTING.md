## How to Contribute

### For Code Changes
```bash
git fork https://github.com/Lycheetah/Lycheetah-Framework
git checkout -b your-feature-branch
# make changes
git commit -m "Clear description of what changed and why"
git push origin your-feature-branch
# open pull request on GitHub
```

### For Mathematics
- Show your work
- Label claims honestly (ACTIVE / SCAFFOLD / CONJECTURE)
- Include worked examples with real numbers
- If you can't compute it, say so — "I don't know" before false confidence

### For Mystery School Content
- Write from experience, not theory
- Include crisis resources where relevant
- Never promise specific outcomes
- Respect the reader's autonomy and intelligence

## Standards

### The Luminous Trinity Test
Every contribution must pass all three simultaneously:

1. **PROTECTOR** — Does this protect? Could it cause harm?
2. **HEALER** — Does this clarify? Does it bypass reality?
3. **BEACON** — Does this illuminate? Does it preserve agency?

If a contribution can't be defended by all three, it doesn't go in.

### Mathematical Honesty
- **ACTIVE** means you can compute it from real inputs and get real numbers
- **SCAFFOLD** means the structure is proven but parameters need measurement
- **CONJECTURE** means it's rigorously formulated but unproven
- Never upgrade a claim's status without evidence
- Never hide a limitation

### Code Standards
- Fail visibly — errors must be detectable and interpretable
- Preserve human agency — no automation that traps users
- Be honest about limits — "I don't know" before false confidence
- UTF-8 everywhere (Windows compatibility matters)
